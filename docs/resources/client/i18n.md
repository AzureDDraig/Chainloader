# I18n and L10n: Translation & Language Path Remappings

ChainLoader features a comprehensive, automated system for remapping, translating, and consolidating language files (`.json` and older `.lang` resources) from legacy Fabric and Forge mods into Minecraft 1.21.1 (NeoForge).

## Runtime Asset Relocation Framework

When legacy mods load, their translation assets are often placed under deprecated folder paths or hardcoded namespaces (e.g., `assets/fabric/lang/en_us.json` or `assets/legacy_compat/lang/en_us.json`). To ensure Minecraft's modern resource manager can find and load these translations, ChainLoader utilizes two primary components:

1. **`VanillaAssetPatcher`**: Scans discovered mod JARs, extracts resources, and registers them into an in-memory `VirtualAssetPack`.
2. **`AssetPathRelocator`**: Performs namespace remapping, path prefix adjustment, and singularization rules on the resource paths on-the-fly.

### The Remapping Process

During the boot phase, `VanillaAssetPatcher.populateModResources()` scans every discovered mod JAR. Zip entry paths are passed to `AssetPathRelocator.getInstance().relocateString(name)`:

```java
String relocatedName = net.chainloader.loader.compat.asset.AssetPathRelocator.getInstance().relocateString(name);
```

For language files, `AssetPathRelocator` relocates:
* **Namespace Remapping**: Map namespaces (e.g., `fabric` -> `chainloader`, `legacy_compat` -> `chainloader`) according to registered rules using `registerNamespaceMapping(source, target)`.
* **Resource Path Mapping**: Ensure paths conform to the modern layout. For example, `assets/fabric/lang/en_us.json` is relocated to `assets/chainloader/lang/en_us.json`.

## Bytecode translation key remapping

Many legacy mods retrieve translation strings by hardcoding registry identifiers or translation keys directly into class files (e.g. `Language.getInstance().getOrDefault("fabric.tooltip.item")`). To solve this, `AssetPathRelocator` acts as a load-time bytecode transformer.

### Constant Pool Scanning (`containsRelocationTargets`)

To minimize overhead, `AssetPathRelocator` runs a fast constant pool pre-scanner before applying heavy ASM parsing. It reads the raw bytecode array and checks if the UTF-8 constants contain any references to mapped namespaces or paths:

```java
public boolean containsRelocationTargets(byte[] classBytes) {
    // ... Checks magic CAFEBABE and reads UTF-8 constants ...
    for (String sourceNamespace : namespaceMappings.keySet()) {
        if (value.contains(sourceNamespace)) {
            return true;
        }
    }
    return false;
}
```

### Bytecode Modification (`AssetRemappingClassVisitor`)

If the pre-scan returns `true`, the bytecode is passed to an ASM-based class visitor (`AssetRemappingClassVisitor`). It intercepts all `LDC` instructions using a custom `MethodVisitor`:

```java
private class AssetRemappingMethodVisitor extends MethodVisitor {
    @Override
    public void visitLdcInsn(Object value) {
        if (value instanceof String) {
            String strValue = (String) value;
            String relocated = relocateString(strValue);
            if (!strValue.equals(relocated)) {
                super.visitLdcInsn(relocated);
                return;
            }
        }
        super.visitLdcInsn(value);
    }
}
```

This ensures that any reference to legacy translation namespaces in the Java bytecode is redirected to the active, consolidated namespace at runtime.

## Language Manager Merging & Virtual Asset Pack

Once the paths are relocated, the raw byte content of the translations is registered to the client-side resources of the `VirtualAssetPack` using a lazy stream supplier:

```java
modPack.registerBytes(
    net.minecraft.server.packs.PackType.CLIENT_RESOURCES, 
    new ResourceLocation(relocatedNamespace, relocatedPath), 
    createJarResourceSupplier(mod.jarFile, originalEntryName)
);
```

During client initialization:
1. Minecraft's vanilla `Language` loader (represented obfuscated as `tw` in 1.21.1) builds the translation map.
2. The virtual asset pack feeds the relocated JSON files to the vanilla asset manager.
3. If multiple mods target the same locale (e.g., `en_us.json`), their keys are merged together in the virtual pack's index, making them completely available to the localized game UI.
